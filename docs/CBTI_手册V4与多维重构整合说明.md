# CBTI 完整手册（V4.0）与多维重构 — 整合说明

本文档结合 **`CBTI_完整手册.docx`**（V4.0）与仓库内抽取稿 **`tools/CBTI_完整手册_extracted.txt`**，回答：维度与题目如何归类、计分/权重/反向题、数据驱动分型、结果解读差异化、多维匹配与映射表，并说明**为何比「纯手工拍区间」更稳**。

---

## 0. 手册规定的核心体系（与实现对齐的锚点）

| 项目 | 手册规定 |
|------|----------|
| 计分 | **A=3 分、B=1 分、C=2 分**；21 题总分 **21（全 B）～63（全 A）** |
| 行为含义 | A：主动/激进/主导；B：被动/保守/服从；C：灵活/折中/回避 |
| 主匹配 | **分数区间命中**：按总分落入 10 个不重叠区间确定人格（见下表） |
| 边界校验 | 当总分落在**区间边界附近**，用手册所称 **「5D 画像余弦相似度」**二次确认，减少错分 |
| 模式串 | 每种人格对应 **21 位 A/B/C 参考串**，用于校验区间与答题一致性 |

**重要：** 当前仓库 `static/quiz-data.json` 中 **题面、选项文案、人格 `patterns` 与总分区间** 与手册 V4.0 **不完全相同**（例如手册 Q1 为「领导急活三选一」，仓库题可能为其他情景）。**若产品声明遵循手册，应以手册题库为准重写或同步 `quiz-data.json`**，否则下列「按 Q1–Q21 编号」的维度权重仅对手册题生效。

## 2. 五维定义（与手册中「典型行为」对齐）

沿用可解释、与手册文案可逐条对照的五维（名称可与产品统一）：

| id | 名称 | 手册侧可对照的典型行为关键词 |
|----|------|-------------------------------|
| drive | 推动力 | 开怼、直电推进、拆饼、多线程、主动报名、主动同步风险 |
| relate | 关系力 | 带实习生、红包群活、团队氛围、教技能、客户沟通 |
| structure | 规则与闭环 | 留记录、交付物写清、便签截止、风险同步、周报实干 |
| boundary | 边界与自我 | 拒无效加班谈钱、周末不卷、不抢风头、温和摸鱼 C、极简工位 |
| pace | 节奏与策略 | 摸鱼 DDL、效率 10%、看状态干活、进攻/咸鱼/伪装型 |

「思考力」在手册题中未单独成大量题干，多分布在 **structure**（审慎、留痕、真话）与 **pace**（策略性摸鱼、伪装）中；若未来加题，可拆出「认知复杂度」子维。

## 4. 选项水平、反向题与关键题加权（让峰谷更明显）

### 4.1 选项 → 水平（0～1）

在手册 **A=3,B=1,C=2** 下，先把分值归一：  
`level = (score - 1) / 2` → B→0，C→0.5，A→1。  
再按第 3 节权重把该题对各维的贡献定为：`contrib[q,d] = w[q][d] * level_q`（可再除以 21 做均值归一）。

### 4.2 建议反向或软惩罚（与手册语义一致）

| 题 | 选项 | 处理 | 理由 |
|----|------|------|------|
| Q17 | C「打工而已开心就好」 | 对 **drive** 做 `1-level` 或对 **structure** 降权 | 与「事在人为」对极 |
| Q19 | A「黑话包装」 | 对 **structure** 乘 0.6～0.7 | 与「只列交付物」对极，手册中流程护法典型行为强调实干周报 |
| Q9 | B「效率降到 10%」 | 可对 **pace** 给「消极抵抗」高分、对 **drive** 低分 | 区分天选牛马与流程护法 |
| Q1 | C「摸鱼到 DDL」 | **pace/回避** 高、**structure** 低 | 与饼饼、天选叙事一致 |

### 4.3 关键题（人格「锚题」）

结合手册 **§3 典型行为**，建议对下列题在算与对应人格相似度时加权 **1.3～1.6×**（易混对共用锚题）：

- **消息黑洞**：Q15（存在感）、Q10（已读不回类）、Q21（极简工位）  
- **流程护法**：Q7、Q19、Q11（一件一件）  
- **天选牛马**：Q9、Q17、Q19（摸鱼/AI 周报）  
- **单机战神**：Q3、Q12、Q9、Q21  
- **群聊气氛组**：Q13、Q3、Q15  
- **饼饼粉碎机**：全书式 C 策略——**Q1、Q2、Q8、Q14** 等「避责/温和抵抗」题权提高  
- **Ctrl+Z**：Q14、Q21、Q16（确认资源）  
- **急急国王**：Q8、Q18、Q11、Q20-A  
- **扛把子**：Q5、Q7-A、Q14-A、Q4-A  
- **锅の传人**：Q1-A、Q2-A、Q7-A、Q20-A  

关键题集合可用 **混淆矩阵**（数据阶段）再精算：找每对人格分歧最大的题位。

## 5. 数据驱动：如何用「典型案例」划边界

1. **标签**：每条样本 =（21 个选项）+ **目标人格**（专家认定，或「与某模式串 Hamming≤k」的弱标签）。  
2. **特征**：`totalScore`；五维向量 `u`；可选「与 10 条手册模式串的 Hamming 向量」。  
3. **区间校准**：在总分上不用硬改区间，可对每类做 **isotonic 回归** 或 **单维阈值学习**，使类内总分方差最小、类间均值差最大。  
4. **5D 质心**：每类样本的 `mean(u)`，与本文档初值做 **收缩融合** `α·数据 + (1-α)·专家`。  
5. **边界规则学习**：用 **有序分类（ordinal）** 或 **XGBoost 多分类** 输出「总分 + 5D」联合规则，替代纯手写边界带宽度。  
6. **验证**：报告 **边界带内错分率**（手册阶段二最关心的指标）。

## 6. 新版匹配逻辑（与手册兼容的「最优拼装」）

**阶段一（与手册一致）：**  
`totalScore = Σ score(option)`，命中上表区间得 **候选人格** `p0`。若总分距相邻区间边界 **> Δ**（建议 Δ=1 或 2 分），直接输出 `p0`。

**阶段二（手册「5D 余弦」）：**  
若 `min(|S - center[p0]|, |S - center[p_neighbor]|) ≤ margin`（如 margin=2），则计算用户 5D 向量 `u` 与 `p0`、`p_neighbor` 的 **质心余弦相似度**，取相似度更高者为最终人格；若仍并列，用 **与两人格参考模式串的 Hamming 较小者** 决胜。

**可选升级（你提出的雷达匹配）：**  
将主匹配改为 **argmax_p cos(u, μ_p)**，总分区间仅作 **先验约束**（落区间外则降权或报警），适合题库大改后「总分与多维不完全单调」的情况。

## 8. 新版「分数 / 向量 ↔ 人格」映射表与「为何更准」

| 人格 | 总分（手册） | 5D 角色（一句话） |
|------|--------------|-------------------|
| 消息黑洞 | 21–24 | 低外显社交、高可靠自闭环 |
| 流程护法 | 25–29 | structure 峰值、冲突回避 |
| 天选牛马 | 30–33 | boundary+pace「清醒抵抗」 |
| 单机战神 | 34–37 | relate 谷、drive/structure 尚可 |
| 群聊气氛组 | 38–41 | relate+pace「气氛与响应」 |
| 饼饼粉碎机 | 42–45 | 多维中等、C 策略拉满「柔性地拒绝」 |
| Ctrl+Z | 46–49 | structure 高 + 纠错快 |
| 急急国王 | 50–53 | drive+pace「推进与急」 |
| 扛把子 | 54–57 | drive+relate 双高、敢担责 |
| 锅の传人 | 58–63 | drive 顶、boundary 表达强 |

**为何比「只按总分落区间」更准？**

1. **同分不同型**：总分相同或差 1 分，5D 形状可完全不同（例如同为 27 分可能是「高结构低社交」或「高边界低结构」）。  
2. **手册已承认边界模糊**：阶段二正是为 **26 分夹在消息黑洞与流程护法之间** 类问题设计；多维是落实手段。  
3. **可学习**：区间与质心可随样本更新，比固定区间更少拍脑袋。  
4. **解读差异化**：结果页除总分外展示 **与最匹配人格的雷达差分**（哪一维最像/最不像），文案可模板化生成。
